# CSitRep Output Template

Use this exact structure for the final report. Fill each section with agent findings.

---

```
============================================================
CONSTRUCTION SITUATION REPORT (CSitRep)
============================================================

PROJECT:  [Project Name]
DATE:     [Today's Date]
PERIOD:   [Weekly/Monthly]
PREPARED: AI-Generated via CSitRep Generator Plugin

============================================================
EXECUTIVE SUMMARY
============================================================

[2-3 sentence overview of overall project health. State the
number of critical issues, watch items, and on-track areas.]

------------------------------------------------------------
CRITICAL ISSUES (Needs Decision Today)
------------------------------------------------------------

[List each critical issue as a numbered item. Include:
 - What the issue is
 - What document/data it came from
 - Impact if not addressed
 - Recommended immediate action]

1. [Issue] - [Source] - [Impact] - [Action needed]
2. ...

------------------------------------------------------------
WATCH ITEMS (Trending Negative)
------------------------------------------------------------

[List items trending negative but not yet critical]

1. [Item] - [Current status] - [Trend direction]
2. ...

------------------------------------------------------------
ON TRACK
------------------------------------------------------------

[Brief summary of areas performing within tolerance]

============================================================
DETAILED STATUS BY DOMAIN
============================================================

------------------------------------------------------------
SCHEDULE STATUS
------------------------------------------------------------

Overall: [CRITICAL / WATCH / ON TRACK]

[Schedule agent findings here]

Key Metrics:
- Planned % Complete: [X%]
- Actual % Complete:  [X%]
- Schedule Variance:  [X days / X%]
- Critical Path:      [Status]

Upcoming Milestones:
- [Milestone] - [Planned Date] - [Status]

------------------------------------------------------------
COST STATUS
------------------------------------------------------------

Overall: [CRITICAL / WATCH / ON TRACK]

[Cost agent findings here]

Key Metrics:
- Total Budget:        $[X]
- Spent to Date:       $[X] ([X%])
- Cost Variance:       $[X] ([X%])
- Burn Rate:           [On track / Over / Under]
- Pending Change Orders: [Count] totaling $[X]

------------------------------------------------------------
SAFETY STATUS
------------------------------------------------------------

Overall: [CRITICAL / WATCH / ON TRACK]

[Safety agent findings here]

Key Metrics:
- Recordable Incidents (Period): [X]
- Open Corrective Actions:       [X]
- Safety Training Completion:    [X%]
- Days Since Last Recordable:    [X]

Open Items:
- [Item] - [Priority] - [Due Date]

------------------------------------------------------------
CONTRACT & ADMIN STATUS
------------------------------------------------------------

Overall: [CRITICAL / WATCH / ON TRACK]

[Contract agent findings here]

Key Metrics:
- Open RFIs:            [X] ([X] overdue)
- Pending Submittals:   [X] ([X] overdue)

------------------------------------------------------------

[Site agent findings here]

Key Metrics:
- Workforce on Site:    [X] workers
- Weather Impact:       [None / Minor / Major]

============================================================
RECOMMENDED ACTIONS
============================================================

[Prioritized list of actions. Each tied to a finding above.]

1. [ACTION] - [Owner] - [By When] - [Related to: Section]
2. ...

============================================================

============================================================

Documents Analyzed:
- Schedule: [list files read]
- Cost: [list files read]
- Safety: [list files read]
- Contracts: [list files read]
- Site: [list files read]

Report generated: [timestamp]
Next report due:  [date based on period]
```
